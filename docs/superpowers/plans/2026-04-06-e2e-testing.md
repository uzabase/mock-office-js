# E2E Testing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add E2E testing that verifies mock-office-js works in a real browser environment within an Excel Add-in application structure.

**Architecture:** Generate an Excel Custom Functions Add-in fixture app using `yo office`, replace the Office.js CDN with mock-office-js via webpack alias, and use Playwright Test to verify mock APIs work correctly through `page.evaluate()`. The dev server from the template serves the fixture, and Playwright's `webServer` option manages its lifecycle.

**Tech Stack:** Playwright Test, webpack (from yo office template), babel-loader with @babel/preset-typescript

**Spec:** `docs/superpowers/specs/2026-04-06-e2e-testing-design.md`

## Task 3: Wire mock-office-js into the fixture app

**Files:**
- Modify: `tests/e2e/fixture/webpack.config.js` (add resolve.alias, switch to HTTP, remove HTTPS certs)
- Modify: `tests/e2e/fixture/src/taskpane/taskpane.html` (replace Office.js CDN script tag)
- Modify: `tests/e2e/fixture/src/taskpane/taskpane.ts` (set up mock globals)
- Modify: `tests/e2e/fixture/src/functions/functions.ts` (use mock-office-js for CustomFunctions)

- [ ] **Step 1: Modify webpack config**

In `tests/e2e/fixture/webpack.config.js`:

1. Add `resolve.alias` to point `mock-office-js` to the library source:

```js
resolve: {
  extensions: [".ts", ".html", ".js"],
  extensionAlias: {
    ".js": [".ts", ".js"],
  },
  alias: {
    "mock-office-js": path.resolve(__dirname, "../../../src"),
  },
},
```

The `extensionAlias` is required because mock-office-js source uses `.js` extensions in imports (e.g., `from "./excel-mock.js"`) per TypeScript ESM conventions. Without it, webpack cannot resolve these to the actual `.ts` files.

2. Switch dev server from HTTPS to HTTP (no need for SSL certs in E2E):

Replace the `devServer` section:

```js
devServer: {
  headers: {
    "Access-Control-Allow-Origin": "*",
  },
  port: process.env.npm_package_config_dev_server_port || 3000,
},
```

3. Remove the `getHttpsOptions` function and the `devCerts` require since we no longer need HTTPS.

4. Remove `office-addin-dev-certs` from the fixture's `package.json` devDependencies.

- [ ] **Step 2: Replace Office.js CDN `<script>` tag in `taskpane.html`**

In `tests/e2e/fixture/src/taskpane/taskpane.html`, replace:

```html
<script type="text/javascript" src="https://appsforoffice.microsoft.com/lib/1/hosted/office.js"></script>
```

with:

```html
<!-- mock-office-js replaces the real Office.js for E2E testing -->
<!-- The mock is bundled by webpack via the taskpane entry point -->
```

(The mock is loaded via the taskpane.ts entry point, not a separate script tag. webpack bundles it.)

- [ ] **Step 3: Modify `taskpane.ts` to set up mock globals and register custom functions**

Replace the contents of `tests/e2e/fixture/src/taskpane/taskpane.ts` with:

```ts
import { ExcelMock } from "mock-office-js";

const mock = new ExcelMock();
(window as any).Excel = mock.excel;
(window as any).CustomFunctions = mock.customFunctions;
(window as any).__mock__ = mock;

// Import functions AFTER globals are set, so associate() calls work
import("../functions/functions");
```

The dynamic `import()` ensures `CustomFunctions` global is available before `functions.ts` calls `CustomFunctions.associate()`. This guarantees correct execution order since `functions.ts` and `taskpane.ts` are separate webpack entry points.

- [ ] **Step 4: Simplify `functions.ts` to use only non-streaming custom functions**

Replace `tests/e2e/fixture/src/functions/functions.ts` with:

```ts
/* global CustomFunctions */

/**
 * Adds two numbers.
 * @customfunction
 * @param first First number
 * @param second Second number
 * @returns The sum of the two numbers.
 */
export function add(first: number, second: number): number {
  return first + second;
}

/**
 * Multiplies two numbers.
 * @customfunction
 * @param first First number
 * @param second Second number
 * @returns The product of the two numbers.
 */
export function multiply(first: number, second: number): number {
  return first * second;
}

// Register custom functions with the mock
CustomFunctions.associate("ADD", add);
CustomFunctions.associate("MULTIPLY", multiply);
```

This removes the streaming functions (clock, increment) that mock-office-js doesn't support, and adds explicit `associate()` calls.

- [ ] **Step 5: Remove `commands.ts` and `functions` entry points from webpack config**

In `tests/e2e/fixture/webpack.config.js`, remove `commands` and `functions` entries (functions.ts is now dynamically imported by taskpane.ts):

- [ ] **Step 4: Write the first failing E2E test — Excel.run reads cell values**

Create `tests/e2e/excel-mock.e2e.test.ts`:

```ts
import { test, expect } from "@playwright/test";

test.beforeEach(async ({ page }) => {
  await page.goto("/taskpane.html");
  // page.goto() waits for 'load' event by default, so mock script has executed
  await page.waitForFunction(() => (window as any).__mock__ !== undefined);
});

test("Excel.run can read cell values set via mock", async ({ page }) => {
  const value = await page.evaluate(async () => {
    const mock = (window as any).__mock__;
    mock.setCell("Sheet1", "A1", 42);

    let result: number[][] = [];
    await (window as any).Excel.run(async (context: any) => {
      const range = context.workbook.worksheets
        .getActiveWorksheet()
        .getRange("A1");
      range.load("values");
      await context.sync();
      result = range.values;
    });
    return result;
  });

  expect(value).toEqual([[42]]);
});
```
